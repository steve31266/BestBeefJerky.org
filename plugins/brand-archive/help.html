Brand Archive Page Plugin - Help Guide

Overview

The Brand Archive Page plugin provides a shortcode [brand_listing] that displays a list of all brand posts from the "brands" custom post type. The list includes:

  • Brand logo (linked to the brand's post)
  • Brand name (linked to the brand's post)
  • Brand headquarters location (if available)
  • Number of reviews associated with the brand

How to Use

  1. Install and activate the plugin.
  2. Insert the shortcode [brand_listing] into a post, page, or template to display the brand listing.
  3. Ensure that the "brands" custom post type exists and includes:
    • ACF Image Field: brands_logo (Return Format: Image URL)
    • ACF Text Field: brands_headquartered
    • Relationship Field: reviews_brand (links "reviews" CPT to "brands" CPT)

Sorting

The listing is sorted alphabetically by Brand Name in ascending order.

File Overview

This plugin consists of the following files:

  • brand-archive.php - The main plugin file that registers the plugin in WordPress. Loads the necessary scripts and styles. Adds the "Help" link to the WordPress Plugins page.
  • brand-listing.php - Defines the shortcode [brand_listing]. Queries the "brands" custom post type. Fetches brand logos, headquarters locations, and associated review counts. Formats the output into a structured list.
  • brand-listing.css - Styles the brand listing layout. Defines the appearance of images, text, and spacing.
  • help.html - This help file, providing instructions on how to use the plugin.
  • images/ (folder, optional) - Stores the placeholder image used when a brand does not have a logo. If missing, ensure the image path in the code is correctly pointing to a valid image.

Help Link

This help file is linked directly from the WordPress Plugins page under the plugin name.

Troubleshooting

  • If the plugin does not appear in the WordPress plugin list, verify that the folder is correctly named brand-archive and contains all necessary files.
  • If the shortcode does not display content, ensure that:
    • The "brands" post type exists.
    • Brands have been created and contain valid ACF data.
    • The "reviews" post type has assigned brands via the "reviews_brand" field.
  • If images do not appear, ensure that the brands_logo field is correctly set to return an Image URL.
  • If review counts are incorrect, check that reviews have properly assigned brand relationships.

Support

For further assistance, contact the developer or check the plugin files in /wp-content/plugins/brand-archive/.
