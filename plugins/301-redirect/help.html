301 Redirects Plugin - Help Guide

Plugin Information

Plugin Name: 301 Redirects

Version: 1.0

Author: Choctaw Websites

Website: https://www.bestbeefjerky.org/

Description

This plugin helps redirect old imported Blogger URLs to their new WordPress structure using 301 permanent redirects. This preserves SEO rankings and ensures users are directed to the correct content.

How It Works

  1. The plugin reads a JSON file (redirects.json) containing old and new URL mappings.
  2. When a user visits an old URL, the plugin checks if a redirect exists.
  3. If a match is found, the user is automatically redirected to the new URL with a 301 redirect.

Installation & Setup

1. Upload & Activate the Plugin

  • Upload the 301-redirect folder to your WordPress /wp-content/plugins/ directory.
  • Go to WordPress Admin > Plugins and activate 301 Redirects.

2. Create and Configure Redirects

  • Open the 301-redirect plugin folder and locate redirects.json.
  • Add your redirects using the following format:
{
    "/2025/02/07/old-post-slug/": "/reviews/new-post-slug/",
    "/2024/12/15/example-old-post/": "/reviews/example-old-post/"
}
  • Each entry maps an old URL (key) to a new URL (value).
  • Ensure the URLs match the path structure of your WordPress site.

3. Testing Your Redirects

  • Visit an old Blogger URL in your browser.
  • The browser should automatically redirect to the correct WordPress URL.
  • If redirects are not working, ensure the redirects.json file is correctly formatted and exists in the plugin directory.

Managing Redirects

  • To add new redirects, update redirects.json with new entries.
  • To remove a redirect, delete the corresponding entry from redirects.json.
  • Changes take effect immediately after saving the file.

Troubleshooting

  • Redirects not working?
    • Check if redirects.json is properly formatted.
    • Make sure the plugin is activated.
    • Clear your browser cache and try again.
